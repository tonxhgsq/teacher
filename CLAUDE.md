# AI 学情工作台 - 项目规范

## 项目定位

教培机构老师的减负工具。核心价值：识别学生薄弱点 → 个性化出题 → 跟进学生情况。

**产品形态**：
- **老师后台**（Web 应用）：老师在电脑上管理学生、出题、看数据
- **飞书 Agent**（家长端）：家长在飞书里和 AI 对话、收反馈、问进度；老师端错题来自系统作业勾选
- 两端共用同一套数据库

**当前阶段**：HTML 原型（验证产品逻辑），下一步迁移为真实 Web 应用

---

## 目标架构（下一阶段）

```
前端：React（复用现有设计系统和交互逻辑）
后端：Node.js + Express
数据库：SQLite（单文件，后期用户多了迁 PostgreSQL）
AI：Claude API（错题分析 + 出题 + 助教对话）
飞书：飞书开放平台 Webhook，接入同一后端
部署：云服务器 + Docker
```

### 数据层现状

- 前端调用后端 API：`http://localhost:3001/api`
- 后端：Node.js + Express
- 数据库：SQLite / libSQL，文件在 `data/teacher.db` 或后端数据目录下
- API 不可用时仍有部分 Mock 降级逻辑
- 当前真实上线前必须补：用户表、班级/机构隔离、积分流水、飞书绑定、审计日志、数据备份

### 后端启动方式

```bash
cd server
node src/index.js
```

或在项目根目录按现有脚本启动本地服务。

### 当前正式上线待办（按优先级）

1. **数据库结构继续稳定**：补正式迁移方案、外键约束、历史数据归属迁移策略。
2. **邮箱验证码发信服务**：当前后端会生成验证码，开发阶段返回给前端展示；上线前需接真实邮件服务。
3. **题库入库链路加固**：字段校验、乱码处理、重复题检测、失败原因提示、删除恢复。
4. **错题分析链路加固**：错题不再依赖照片 OCR，老师从已生成作业中标记错题，AI 只给薄弱点建议，最终由老师确认。
5. **积分后端化**：AI 分析、作业生成、题库分类统一走后端扣费接口，余额和流水刷新后仍准确。
6. **飞书真实验收**：扫码创建机器人后能被拉群，群成员 @ 能回复，老师可开关群内回复。
7. **部署与安全**：HTTPS、环境变量、日志、备份、监控、错误报警、密钥不进前端。
8. **核心流程测试**：注册登录、导入题库、识别错题、生成薄弱点、生成作业、积分扣费、飞书消息。

## 数据结构

```javascript
// 学生
{ id, name, grade, status,        // status: high-risk/follow-up/improving/stable
  weakPoints[],                    // 薄弱知识点（字符串数组）
  errorCauses{},                   // { 审题类: n, 计算类: n, ... }
  homeworkRate, lastRecord,
  recentErrors[], notes, suggestion }

// 题目
{ id, content, type, difficulty, knowledgePoint, answer, status }
// status: pending-ocr / pending-review / approved

// 知识点掌握状态
studentNodeStatus = {
  studentId: { nodeId: 'weak'|'partial'|'mastered'|'untested' }
}
```

## 代码规范（HTML 原型）

- Tab 内函数加前缀：`mistake*` / `km*` / `hw*`
- 全局状态：`currentStudent`、`currentTab`
- 操作反馈统一用 `showToast(msg)`
- 每个 Tab 对应 `render{TabName}()`，切换时调用
